# 旅行计事本 — 设计文档

日期：2026-07-13

## 背景与目的

用户想做一个免费、可通过链接打开的网页，用来给男朋友展示一次具体的旅行计划（行程、路线、预算等），并让他能做一些轻量互动（选心情、留言、批准预算）。这次先只承载**一条确定的行程**（不做多方案选择）。

## 目标用户 / 使用场景

- 制作者（用户本人）：偶尔更新一次内容，找 Claude 帮忙改代码并重新部署，不需要自己写代码。
- 查看者（男朋友）：用手机浏览器打开一个链接即可查看，无需安装任何东西、无需登录。

## 技术架构

- **纯静态网页**：HTML/CSS/原生 JS，不使用前端框架，不需要构建工具（build step）。
- **数据与展示分离**：行程内容（每日安排、预算明细、目的地卡片、路线节点等）集中放在一个独立的数据文件（`data.js`）中，页面结构/样式代码不需要因为改内容而改动。
- **独立仓库**：新建独立的 git 仓库 `travel-plan`（不与用户现有的 `date` 仓库中其他无关项目混在一起）。
- **部署**：Vercel 免费版，关联 GitHub 仓库后自动部署；每次更新内容 push 到 GitHub 即可自动生效，链接保持不变。
- **表单收集**：男朋友的"心情选择"和"留言"通过 Formspree 免费表单服务提交，自动发邮件到用户邮箱（renata.rt.1027@gmail.com），用户无需额外操作、无需自建后端。

## 视觉风格

手绘旅行手账风格：暖色调、手绘感线条与路线图，整体观感像一本旅行笔记本而非商务网站。默认按手机竖屏优先做响应式布局。

## 页面内容模块

1. **封面**：旅行标题、封面图、一句引子文案。
2. **按天行程时间线**：可点开/收起每一天，展示当天的时间、地点、交通方式与活动安排。
3. **手绘风格路线图**：用 SVG 手绘线条连接行程中的城市/地点节点，不接入任何地图 API，保证稳定加载、无需 API Key。
4. **预算明细**：按类别（交通/住宿/餐饮/门票等）列支出，并显示总额；附一个"批准预算"按钮，点击后通过 Formspree 提交一条确认消息。
5. **目的地/照片介绍卡片**：每个目的地一张卡片，含图片和简介，点击可展开查看更多文字。
6. **心情选择 + 留言板**：一组心情表情按钮 + 一个文字留言框，提交后都通过 Formspree 发送到用户邮箱。

## 数据结构（草案）

`data.js` 大致包含：

- `trip`: 标题、封面图、引子文案
- `days`: 数组，每项含日期、地点、时间线条目列表（时间/活动/交通方式/备注）
- `route`: 路线节点数组（城市名 + 相对坐标，用于 SVG 连线绘制）
- `budget`: 分类数组（类别名 + 金额），加总额
- `destinations`: 数组，每项含图片、名称、简介

具体行程内容由用户之后提供，先用占位数据搭建结构。

## 错误处理

- Formspree 提交失败（网络问题等）：前端展示"发送失败，请稍后重试"提示，不阻塞页面其他功能。
- 无地图 API、无后端数据库依赖，避免了对应的加载失败/密钥失效风险。

## 测试方式

- 本地用浏览器直接打开静态文件 / 本地预览服务器，手动检查各模块交互（时间线展开、路线图渲染、预算按钮、留言提交）。
- 检查移动端窄屏布局是否正常。
- 实际提交一次表单，确认邮件能收到。
